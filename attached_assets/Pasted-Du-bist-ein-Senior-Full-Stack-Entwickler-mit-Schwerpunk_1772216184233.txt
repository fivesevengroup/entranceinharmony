Ausgangslage
Ich habe eine Single Page Application, die aktuell nur <div id="root"></div> im initialen HTML liefert und den Content erst per JavaScript rendert. Das ist schlecht für SEO, weil Google beim ersten Crawl kaum Inhalt sieht.

Ziel
Baue eine Lösung, die für Suchmaschinen sofort fertiges HTML ausliefert.
Priorität 1: Server Side Rendering oder Pre Rendering für Startseite und alle wichtigen Landingpages.
Priorität 2: Saubere Meta Tags pro Route inklusive title, description, canonical, hreflang, og und twitter.
Priorität 3: Strukturierte Daten pro Seite (Organization, LocalBusiness, Service).
Priorität 4: Performance (schneller LCP, schlankes HTML, komprimierte Assets).

Technischer Rahmen
Projekt ist mit Vite + React gebaut (falls du ein anderes Setup erkennst, passe die Lösung entsprechend an).
Ich deploye über Replit. Lösung muss in Replit laufen und buildbar sein.
